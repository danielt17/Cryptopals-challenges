# Contributing

Thanks for taking the time to contribute!

## Scope
- Keep changes focused on documentation, structure, or challenge implementations.
- Avoid changing algorithms unless a fix is required.

## Workflow
1. Create a feature branch.
2. Make changes with clear, minimal commits.
3. Run `python scripts/verify_project.py` before opening a PR.

## Style
- Keep documentation concise and consistent with existing writeups.
- Use ASCII unless the file already contains Unicode.
